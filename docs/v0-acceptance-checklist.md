# Resume Builder Current Acceptance Checklist

Date: 2026-07-10

## Prerequisites

在项目目录运行：

```bash
cd ~/Downloads/resume-builder
npm install
```

如果首次运行缺少 Chromium：

```bash
npx playwright install chromium
```

如果需要检查 PDF 页数但缺少 `pdfinfo`：

```bash
brew install poppler
```

## Automated Check

运行：

```bash
npm test
```

期望：

- 所有测试通过。
- 当前测试数量为 57。

## Editor Startup

运行：

```bash
npm run editor
```

期望：

- 终端输出本地地址，例如 `http://127.0.0.1:4321`。
- 如果 `4321` 被占用，会自动尝试后续端口。
- 浏览器打开该地址后，左侧显示 A4 HTML 预览，右侧显示编辑器。
- A4 默认完整显示在左侧预览区内，浏览器外层页面不需要纵向滚动。
- 右侧顶部导航、底部操作栏和状态提示保持可见，只有中间表单区域滚动。

## Preview Linking

步骤：

1. 进入 `内容编辑`。
2. 点击左侧预览里的姓名、求职意向、学校、专业、电话或邮箱区域。
3. 点击左侧预览里的照片。
4. 点击左侧预览里的某一段 `实习经历`、某个 `专业技能` 技能组、某个 `项目经历` 卡片。
5. 点击左侧预览里的公司、时间、角色、概述、技能标题或 bullet 文本。

期望：

- 鼠标悬停时，左侧具体字段有轻量高亮，父级 section 或卡片不跟着变蓝。
- 点击后，左侧模块保持选中态。
- 点击基本信息字段时，右侧自动切到 `基本信息`，并聚焦对应输入框。
- 点击照片时，右侧自动切到照片模块。
- 点击实习经历、专业技能、项目经历卡片时，右侧自动切到对应内容模块，并定位到对应卡片。
- 点击实习经历、专业技能、项目经历里的具体字段或 bullet 时，右侧自动定位并聚焦到对应输入框。
- 底部状态显示 `正在编辑：...`。
- 点击预览不会直接修改简历内容。

## Add Item Focus

步骤：

1. 进入 `内容编辑`。
2. 在 `实习经历` 点击 `新增`。
3. 在 `项目经历` 点击 `新增`。
4. 在 `专业技能` 点击 `新增`。
5. 在某个技能组或经历卡片里点击 `新增要点`。

期望：

- 新增实习经历后，右侧自动聚焦该经历的 `开始` 输入框。
- 新增项目经历后，右侧自动聚焦该项目的 `开始` 输入框。
- 新增技能组后，右侧自动聚焦该技能组的 `标题` 输入框。
- 新增要点后，右侧自动聚焦新建的 bullet textarea。
- 新增后状态显示 `未保存` 和 `预览未更新`。

## Delete Confirmation

步骤：

1. 进入 `内容编辑`。
2. 在 `专业技能`、`实习经历` 或 `项目经历` 中点击某个 `删除`。
3. 点击出现的 `取消`。
4. 再次点击同一个 `删除`，然后点击 `确认删除`。

期望：

- 第一次点击 `删除` 不会立刻删除内容。
- 当前卡片或要点下方出现独立的删除确认条，不弹出浏览器确认框。
- 点击 `取消` 后，内容保持不变，按钮恢复为普通 `删除`。
- 点击 `确认删除` 后，只删除当前行或当前 bullet。
- 删除后右侧自动聚焦相邻内容；如果列表已空，则聚焦对应的新增按钮。
- 删除后状态显示 `未保存` 和 `预览未更新`。

## Footer Actions

步骤：

1. 将编辑器窗口缩小到约 1024px 宽。
2. 查看右侧编辑栏底部的样例、备份和生成操作。

期望：

- 样例与备份操作在左侧上下排列。
- `生成 PDF` 或 `重新生成预览` 固定在右侧，不会被挤到辅助操作下方。
- 下拉框和按钮之间没有重叠或裁切。
- 整张 A4、底部操作栏和状态提示同时位于浏览器视口内。

## Basic Content Edit

步骤：

1. 进入 `内容编辑`。
2. 打开 `基本信息`。
3. 临时修改姓名或求职意向。
4. 点击 `保存`。
5. 点击 `生成 PDF`。

期望：

- 修改后状态显示 `未保存` 和 `预览未更新`。
- 修改后生成按钮显示 `保存后生成` 且不可点击。
- 点击旧预览时，底部提示这是上一次生成的预览。
- 保存后状态显示 `已保存` 和 `待生成`。
- 保存后生成按钮显示 `生成 PDF`。
- 生成成功后状态显示 `预览已更新`。
- 生成成功后生成按钮显示 `重新生成预览`。
- `output/resume.pdf` 内容同步变化。
- `resume.yaml` 中内容已更新。
- 保存前的旧版本存在于 `resume.backup.yaml`。
- 保存前的旧版本也存在于 `backups/resume-*.yaml`。

## Photo Replacement

步骤：

1. 进入 `内容编辑`。
2. 打开 `照片`。
3. 上传一张 `.png`、`.jpg`、`.jpeg`、`.webp` 或 `.svg`。
4. 点击 `生成 PDF`。

期望：

- 照片路径更新到 `assets/photo.*`。
- 左侧预览和 PDF 使用新照片。
- 上传前的 `resume.yaml` 已备份为 `resume.backup.yaml`。
- 上传前的 `resume.yaml` 也已备份到 `backups/resume-*.yaml`。
- 超过大小限制或不支持格式时，界面显示错误，不覆盖当前内容。

## Section Order

步骤：

1. 打开顶部 `排版顺序`。
2. 使用 `上移` / `下移` 调整 `实习经历`、`专业技能`、`项目经历` 的顺序。
3. 点击 `保存`。
4. 点击 `生成 PDF`。

期望：

- `resume.yaml` 的 `layout.sectionOrder` 同步变化。
- PDF 和 PNG 的 section 顺序同步变化。
- 基本信息和照片仍固定在页首。

## Examples

步骤：

1. 在底部样例下拉框选择一个样例。
2. 点击 `载入样例`。

期望：

- 当前 `resume.yaml` 被样例内容替换。
- 替换前自动生成 `resume.backup.yaml`。
- 替换前也自动生成 `backups/resume-*.yaml`。
- 载入后需要重新点击 `生成 PDF` 才会更新左侧预览。

## Versioned Backups

步骤：

1. 在网页中保存一次内容，或替换照片/载入样例。
2. 查看底部 `最近备份` 下拉框。
3. 选择一份备份，点击 `恢复备份`。
4. 在确认框中确认恢复。

期望：

- 每次覆盖 `resume.yaml` 前都会生成 `resume.backup.yaml`。
- 每次覆盖 `resume.yaml` 前也会生成一份 `backups/resume-YYYYMMDD-HHMMSS.yaml`。
- `最近备份` 下拉框显示最近的时间戳备份。
- 点击 `恢复备份` 前会弹出确认框。
- 确认后，`resume.yaml` 恢复为所选备份内容。
- 恢复前的当前内容会再次备份，避免恢复操作本身造成不可逆覆盖。
- 恢复后状态显示 `已保存` 和 `待生成`。

## Operation Safety

步骤：

1. 修改一个字段，让 `保存` 按钮可点击。
2. 点击 `保存`。
3. 保存过程中观察底部操作区。
4. 如果浏览器加载了新版前端但服务端未重启，刷新页面后查看底部提示。
5. 使用缺少字段定位标记的旧版 `output/preview.html` 打开编辑器。

期望：

- 保存请求未完成时，保存按钮显示 `保存中`。
- 保存请求未完成时，`保存`、`生成 PDF`、`载入样例`、`恢复备份` 不可重复点击。
- 如果前端和后端接口不匹配，底部提示会明确要求重新运行 `npm run editor`。
- 如果预览由旧版本生成，底部提示会要求重新生成 PDF/预览，状态显示 `待生成`。

## Visual QA

打开 `output/resume.png` 或查看左侧预览：

- 没有文字被裁切。
- bullet 没有显示异常方框。
- 链接没有横向溢出。
- 照片固定在右上角。
- 简历内容保持一页 A4。
- section 顺序符合 `layout.sectionOrder`。
- 编辑器默认完整显示整张 A4，不产生浏览器外层纵向滚动。

## Recovery

如果误保存、误替换照片、误载入样例或误删内容：

1. 在网页底部 `最近备份` 下拉框选择一份备份。
2. 点击 `恢复备份`。
3. 确认恢复。
4. 点击 `生成 PDF`。

期望：

- `resume.yaml` 恢复到备份内容。
- 输出文件重新生成后与恢复内容一致。

## Known Non-Checks

v0 验收不检查：

- Word 导出。
- 多模板切换。
- 多页简历。
- AI 改写。
- 精确 overflow 定位到某一条 bullet。

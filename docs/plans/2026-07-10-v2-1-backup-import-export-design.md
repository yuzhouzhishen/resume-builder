# V2.1B 简历数据导入导出设计

日期：2026-07-10

状态：V2.1B 已实施并验证。自动化测试共 156 项通过；真实数据只读导出和临时目录导入往返、桌面与 1024px 视口浏览器检查均已完成。

## 目标

V2.1B 为本地简历编辑器增加整套数据导出与导入，使用户换电脑时不需要手动寻找 `resumes.json`、YAML、照片和备份目录。功能入口位于编辑器顶部工具栏，导出使用普通 ZIP，导入采用“检查后确认”，并在替换当前数据前自动保留完整的导入前目录。

V2.1B 不实现云同步、账号系统、跨设备自动发现、密码加密、单份简历合并导入或 PDF/图片识别。

## 已确认决策

- 导入语义为整套替换，不合并简历 ID、名称或照片。
- 导入前自动保留当前整个 `dataRoot`，不覆盖已有备份。
- 导出包包含简历、照片和备份，不包含可重新生成的 `output/`。
- 导出包为不加密的普通 ZIP，UI 必须提示其中含个人信息。
- 入口为顶部工具栏中的“数据管理”菜单，不新增独立页面。
- 导入分为检查和提交两个阶段，未确认前不修改正式数据。
- 存在未保存草稿时禁止导入，用户必须先保存或撤销。

## ZIP 格式

默认文件名：

```text
resume-builder-backup-YYYYMMDD-HHMMSS.zip
```

ZIP 根目录固定为：

```text
manifest.json
resumes.json
resumes/
assets/
backups/
```

以下内容不进入导出包：

- `output/` 下的 HTML、PDF 和 PNG。
- `.env.local` 和任何本机绝对路径配置。
- `.migration.json`、`.migration-in-progress` 和导入临时文件。
- 旧版根级 `resume.backup.yaml`。
- 代码、模板、公开样例和 `node_modules/`。

`resumes/`、`assets/` 和 `backups/` 中只导出常规文件与目录，拒绝符号链接和其他特殊文件。空目录不作为数据契约；导入后按需重新创建。

## Manifest

`manifest.json` 使用 UTF-8 JSON，格式版本从 `1` 开始：

```json
{
  "format": "resume-builder-backup",
  "formatVersion": 1,
  "createdAt": "2026-07-10T15:30:00.000Z",
  "appVersion": "0.1.0",
  "activeResumeId": "resume-b",
  "resumeCount": 2,
  "resumes": [
    { "id": "cpp", "name": "C++ 应届生" }
  ],
  "files": [
    {
      "path": "resumes.json",
      "size": 320,
      "sha256": "..."
    }
  ]
}
```

`files` 按路径排序，列出除 `manifest.json` 外的每个文件。导入时要求 ZIP 中的文件集合与清单完全一致；缺失、额外、重复、大小不符或 SHA-256 不符都视为损坏包。

`activeResumeId`、`resumeCount` 和 `resumes` 是检查界面的摘要字段，但最终仍以包内经过验证的 `resumes.json` 为准。摘要与注册表不一致时拒绝导入。

## 技术选型

新增 npm 依赖 `fflate`，在 Node 内完成 ZIP 编解码。不调用 macOS 的 `zip`、`unzip` 或其他系统命令，保证换电脑和不同操作系统下行为一致。

V2.1B 数据量较小，可以在内存中构建导出 ZIP。导入仍设置独立限制：

- ZIP 请求体最大 50 MB。
- 解压后总大小最大 100 MB。
- 文件数量最大 2,000。
- 单文件最大 20 MB。
- 压缩率和累计大小在写入临时目录前检查。

服务端拒绝绝对路径、空路径、`.`、`..`、反斜杠路径、NUL、重复规范化路径、未知顶层路径和目录外解析结果。解压数据只能写入当前导入会话拥有的临时目录。

## 导出流程

`GET /api/data/export` 执行：

1. 调用现有 `validateDataRoot()` 校验当前正式数据。
2. 枚举允许范围内的常规文件，拒绝符号链接。
3. 为每个文件计算大小和 SHA-256。
4. 生成 `manifest.json` 并构建 ZIP。
5. 返回 `application/zip` 和 `Content-Disposition: attachment`。

ZIP 直接作为响应返回，不写入代码仓库或 `dataRoot`。导出是只读操作，不修改当前简历、备份、预览或活动 ID。

## 两阶段导入

### 检查阶段

浏览器选择 ZIP 后，以原始 `application/zip` 请求体调用 `POST /api/data/import/inspect`。服务端：

1. 限制压缩请求体大小。
2. 解析 ZIP，并执行路径、文件数和解压后大小检查。
3. 校验 `manifest.json`、文件集合、大小和 SHA-256。
4. 在 `dataRoot` 同级创建 `.resume-import-<token>` 临时目录。
5. 只向该临时目录写入允许文件。
6. 调用 `validateDataRoot()` 校验注册表、全部 YAML、照片和真实路径边界。
7. 返回随机 token、创建时间、格式版本、活动简历和简历列表。

一个服务器进程只保留一个待确认导入。检查新文件会清理该进程拥有的上一份未发布临时目录。token 默认 15 分钟过期；取消、过期或服务退出时只清理未发布目录。服务重启后遗留的临时目录不自动当作正式数据，也不触碰当前 `dataRoot`。

### 提交阶段

用户确认后调用 `POST /api/data/import/commit`，请求体为 `{ "token": "..." }`。服务端先验证 token、有效期和临时目录所有权，再进入导入锁定状态。

导入提交请求到达后，保存、备份恢复、上传照片、生成、创建、重命名、切换和删除简历等新写操作返回 HTTP 423。已有写操作尚未结束时，提交本身返回 423 且保留 token 供重试；客户端断开不会提前释放仍在后台运行的写操作。读取和静态资源可继续完成，但 UI 会显示阻塞状态，避免用户继续操作。

正式切换使用同一父目录中的重命名：

```text
resume-builder-data
→ resume-builder-data.pre-import-YYYYMMDD-HHMMSS

.resume-import-<token>
→ resume-builder-data
```

第二次重命名失败时，立即把旧目录改回 `dataRoot`。新目录发布后再次调用 `validateDataRoot()`；复核失败时，将新目录退回隔离路径并恢复旧目录。任何失败都不得递归删除当前正式数据或导入前备份。

成功后保留 `resume-builder-data.pre-import-*`，不自动删除。响应返回活动 ID、简历列表和备份目录名称。导入包没有 `output/`，因此正式 PDF 状态为未生成，页面先显示实时 HTML 草稿。

## HTTP API

新增三个接口：

- `GET /api/data/export`：返回 ZIP 下载。
- `POST /api/data/import/inspect`：接收原始 ZIP，返回检查摘要和 token。
- `POST /api/data/import/commit`：确认整套替换。

可选增加 `DELETE /api/data/import/:token` 用于用户取消后立即清理待确认目录；即使浏览器没有调用，过期机制仍必须生效。

接口错误保持 JSON 结构 `{ ok: false, error }`。ZIP 下载失败返回普通 JSON 错误，不返回半个成功压缩包。

## UI 设计

在顶部简历选择器旁增加带数据库图标的“数据管理”按钮。菜单仅包含：

- 导出数据包。
- 导入数据包。

导出前显示隐私确认：“数据包包含姓名、联系方式和照片，且未加密。”确认后触发浏览器下载，成功状态显示导出的简历数量。

导入对话框包含三个稳定状态：

1. 选择文件：显示文件名和大小。
2. 检查完成：显示创建时间、版本、简历数量、活动简历和名称列表。
3. 确认替换：显示当前数据会被整体替换，以及旧数据会自动保留。

有未保存草稿时禁止打开文件选择器，并把焦点引导到现有保存区域。提交期间关闭按钮和重复提交按钮不可用，显示“正在备份并导入”。完成后关闭菜单与对话框，重新读取 `/api/resumes`、活动 YAML、备份列表和草稿预览。

导入前备份位置显示为可复制的普通文本，不在界面提供删除按钮。V2.1B 第一版不实现备份目录管理和一键回退；保留目录可由后续版本接管。

## 错误处理

用户可见错误按阶段区分：

- ZIP 超过限制、不是 ZIP 或无法解压。
- 格式版本不支持。
- 清单缺失、结构无效或摘要与注册表不一致。
- 文件缺失、多余、重复、大小或哈希不一致。
- ZIP 路径非法或包含符号链接/特殊文件。
- 简历 YAML、照片或注册表无效。
- 待确认 token 无效或已过期。
- 当前数据不可写、备份重命名失败。
- 正式切换失败，旧数据已恢复。
- 正式切换失败且需要人工检查隔离目录。

错误信息不得回显简历正文、联系电话、邮箱或照片字节。

## 测试策略

新增独立数据包模块测试：

- 多份简历、照片和备份导出后可完整导入。
- 文件顺序稳定，清单 SHA-256 正确。
- `output/`、`.env.local` 和迁移文件不进入 ZIP。
- 符号链接、绝对路径、`..`、反斜杠、重复路径和未知顶层路径被拒绝。
- ZIP、单文件、总解压大小和文件数量限制生效。
- 缺失文件、额外文件、损坏哈希和不支持版本被拒绝。
- 临时目录通过现有 `validateDataRoot()` 后才能返回 token。
- token 过期、取消和重复提交不会修改正式数据。
- 导入锁阻止并发写操作。
- 第二次重命名失败时恢复旧目录。
- 发布后复核失败时保留隔离目录并恢复旧数据。
- 导入前备份目录不被覆盖或删除。

编辑器测试覆盖菜单位置、隐私提示、文件选择、检查摘要、未保存草稿拦截、提交中状态、成功后重新加载和错误展示。

真实验收只对当前私人数据执行导出；导入测试使用导出包的副本和临时 `dataRoot`，不直接替换真实数据目录。最后确认导出和临时导入不会让 Git 工作区变脏。

## 非目标

V2.1B 第一版不做：

- 密码加密或密钥恢复。
- 单份简历导入、合并或冲突解决。
- 导入 PDF、Word、PNG 或第三方简历格式。
- 自动上传网盘、云同步或定时备份。
- 导出 `output/` 生成结果。
- 自动删除导入前备份目录。
- 可视化管理或一键恢复历史导入前备份。

# V1 实时草稿预览设计

日期：2026-07-10

状态：已实现并完成自动化与视觉验收

## 目标

用户修改右侧内容后，左侧 A4 HTML 预览在停止输入约 300ms 后自动更新，不再要求先保存、再生成 PDF 才能观察文字和排版变化。

实时预览只处理浏览器中的草稿，不改变以下边界：

- 不自动写入 `resume.yaml`。
- 不自动创建备份。
- 不自动生成 PDF 或 PNG。
- `保存` 仍然是持久化内容的唯一入口。
- `生成 PDF` 仍然是正式选择密度和导出文件的唯一入口。

## 方案

服务端新增 `POST /api/preview`。请求体包含当前内存中的 `resume`，服务端执行与保存、正式生成相同的结构验证，然后复用 `renderResumeHtml()` 返回 HTML。该接口不写入任何文件。

前端在内容、bullet、经历、技能组或排版顺序改变后启动 300ms 防抖。最后一次请求成功后，通过 iframe `srcdoc` 显示草稿 HTML。快速连续输入产生的新版本号会使旧响应失效，避免旧内容覆盖新内容。

## 密度适配

草稿 HTML 首先以 `normal` 渲染。iframe 加载后，前端使用与正式生成相同的内容边界测量方式，依次尝试：

1. `normal`
2. `compact`
3. `tight`

找到能放入一页 A4 的第一档后更新状态栏密度。若 `tight` 仍溢出，预览保留 `tight`，同时显示超出像素和缩短内容建议。正式 PDF 生成仍使用 Playwright 再次独立测量，因此草稿测量不是导出承诺。

## 状态语义

- `预览更新中`：草稿请求或 iframe 渲染尚未完成。
- `草稿预览`：左侧显示当前未保存内容。
- `草稿预览失败`：当前草稿未通过验证或接口请求失败，左侧保留上一次可用预览。
- `草稿超出 A4`：已使用 `tight`，但内容仍超过一页。
- `PDF 待生成`：内容已保存，但正式 PDF 还没有重新生成。
- `预览已更新`：正式 PDF、PNG 和 HTML 已生成。

点击草稿预览中的字段仍然定位右侧表单。提示文案使用“当前为未保存草稿”，不再误报为“上一次生成的预览”。

## 错误处理

- 必填字段暂时为空时，草稿接口返回验证错误，不保存内容。
- 快速输入时只采用最后一次请求结果。
- 预览接口沿用 5MB JSON 请求体限制。
- 服务端版本过旧、没有 `/api/preview` 时，沿用前后端版本不一致提示。
- 正式生成开始时取消待处理草稿，并切回 `output/preview.html`。

## V1 第一阶段不做

- 自动保存。
- 每次输入自动生成 PDF。
- 自定义缩放控件。
- 多份简历管理。
- 可视化字号、边距或间距调节。

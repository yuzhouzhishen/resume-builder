# Resume Builder V2.2 CI And Privacy Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a local and GitHub CI quality gate that rejects private resume data and runs the complete test suite on every change.

**Architecture:** Implement a dependency-free Node.js scanner with pure path/content rules plus Git-backed current-tree and history inspection. Wire it into the existing npm test entry point, then add a minimal GitHub Actions workflow that installs Chromium and CJK fonts before running the same command.

**Tech Stack:** Node.js ESM, Node test runner, Git CLI, js-yaml, GitHub Actions, Playwright Chromium.

### Task 1: Specify Privacy Rules With Failing Tests

**Files:**
- Create: `scripts/privacy-check.test.mjs`
- Create: `scripts/privacy-check.mjs`

1. Add tests for allowed public examples and forbidden runtime paths, raster/document/archive files, personal emails, mobile numbers, absolute home paths and token patterns.
2. Build sensitive test fixtures from string fragments so the test source does not itself violate the scanner.
3. Run `node --test scripts/privacy-check.test.mjs` and confirm it fails because the scanner exports do not exist.
4. Implement `findPathViolations()` and `findTextViolations()` with deterministic rule IDs and redacted messages.
5. Run the focused tests and confirm they pass.

### Task 2: Scan The Current Tree And Full Git History

**Files:**
- Modify: `scripts/privacy-check.mjs`
- Modify: `scripts/privacy-check.test.mjs`

1. Add a temporary-repository integration test that commits a forbidden file, removes it in a later commit, and expects history scanning to fail.
2. Add tests for unsafe commit emails and for a clean temporary history.
3. Run the focused tests and confirm the new integration tests fail.
4. Implement tracked-path inspection with `git ls-files`, unique historical blob inspection with `git rev-list --objects --all` and `git cat-file`, and author/committer email inspection with `git log`.
5. Add a CLI that prints concise violations and exits `1`, or prints a success summary and exits `0`.
6. Run `node --test scripts/privacy-check.test.mjs` and `node scripts/privacy-check.mjs`; both must pass in the clean worktree.

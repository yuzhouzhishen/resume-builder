# Resume Builder V2.3 Acceptance Checklist

Date: 2026-07-11

V2.3 继续执行数据隔离、多简历、一页 A4、实时草稿、显式保存、单份简历备份、导入导出和隐私检查，并增加整套历史数据恢复验证。

## Automated Verification

```bash
npm test
```

预期全部测试通过，无失败、取消或跳过；不要在文档中固定会随实现增长而变化的测试数量。结果应覆盖：

- [ ] 隐私扫描、CI workflow、路径解析、外部数据根迁移和注册表校验。
- [ ] 快照命名、两种类型、排序、无效树、固定无路径错误和不透明 ID。
- [ ] staging 精确复制、源快照变化检测、原子发布、最终校验、唯一恢复前备份、隔离和回滚。
- [ ] 单编辑器进程边界、同服务器导入/恢复替换门禁和普通写入并发；不创建跨进程锁文件。
- [ ] 恢复 API、终态与暂时错误映射、提交后仅重试界面刷新，以及 API 不泄露路径。
- [ ] 恢复中心的加载、空列表、有效/无效记录、详情、确认、单次提交、异步代次保护、焦点和响应式布局。
- [ ] 导入导出、单份简历备份、编辑器、草稿、照片、渲染和正式生成回归。

复制失败、发布失败、最终校验失败、隔离失败和回滚失败必须依靠临时目录与注入故障的自动化测试验证。不要在个人运行数据上手动破坏文件、权限或事务来制造这些失败。

## Safe Setup

- [ ] 关闭所有指向个人数据目录的编辑器进程；同一个测试 `dataRoot` 也只启动一个编辑器进程。
- [ ] 为人工验收创建全新的仓库外临时目录，并只写入虚构姓名、占位联系方式和测试照片。
- [ ] 通过合成数据导入或复制合成夹具创建 `pre-import`、`pre-restore`、有效和无效候选；不得从个人运行数据制作测试快照。
- [ ] 启动时显式指定该临时目录，例如：

```bash
manual_root="$(mktemp -d)/resume-builder-data"
RESUME_BUILDER_DATA_DIR="$manual_root" npm run editor
```

验收结束后只处理这个可确认的临时根；不要把 ZIP、快照、生成输出或临时目录加入 Git。

## Recovery Center States

- [ ] `数据管理` 菜单按 `导出数据包`、`导入数据包`、`恢复历史数据` 排列。
- [ ] 没有快照时显示明确空状态，不显示或启用恢复操作。
- [ ] 有效和无效快照都按时间倒序列出，并明确显示 `可恢复` / `不可恢复` 状态。
- [ ] 无效记录显示固定、无路径原因，不能选择；有效记录可选择并显示类型、时间、简历数量、活动简历和名称详情。
- [ ] 键盘可以进入列表并操作有效记录；焦点样式可见，打开、切换状态、关闭和完成后的焦点位置合理。
- [ ] 长名称正常换行，列表内部滚动；代表性的宽屏和窄屏视口中，对话框、状态、详情和操作区无横向溢出或遮挡。
- [ ] 列表读取失败显示安全提示和 `重试`；关闭后重新打开时，旧异步响应不能覆盖新列表。

## Draft And Restore Matrix

- [ ] 有未保存草稿时，从菜单打开恢复中心会被阻止并聚焦 `保存`。
- [ ] 打开后再产生未保存草稿时，最终确认仍会阻止提交。
- [ ] 有效 `pre-import` 快照可以恢复；界面应用恢复 API 返回的注册表，再读取活动简历及其单份简历备份，全部简历 YAML、照片、备份和活动简历与快照一致。
- [ ] 有效 `pre-restore` 快照可以恢复，结果与该快照一致。
- [ ] 同一个源快照连续恢复两次均成功，源快照仍存在且内容不变。
- [ ] 每次恢复成功后重新打开恢复中心，提交前的当前完整数据以新的唯一 `pre-restore` 记录出现，可用于撤销本次恢复。
- [ ] 确认恢复期间关闭、取消和重复提交被禁用，只发送一次恢复请求。
- [ ] 恢复成功后读取活动简历及其单份简历备份，关闭恢复对话框，再由现有预览流程异步安排草稿渲染；正式输出状态明确为 `PDF 待生成`，且没有自动请求 PDF、PNG 或 HTML 生成。
- [ ] 即使快照物理带回了旧 `output/<id>/` PDF、PNG 或 HTML，界面也不把它们视为当前输出，显式生成前保持 `PDF 待生成`。
- [ ] 恢复后显式点击 `生成 PDF`，新 PDF、PNG 和 HTML 才与恢复后的内容一致。

## Error And Retry Behavior

- [ ] 选中的快照已删除、找不到或重新校验无效时，界面重新读取列表、清除选择并要求重新选择，不重复提交旧 ID。
- [ ] 同一服务器内恢复已在进行、写入门禁或其他暂时错误保留确认详情，允许稍后重试同一恢复。
- [ ] 确认不会创建跨进程锁文件，也不存在过期锁或手动清锁流程。
- [ ] 复制、验证或发布前失败时，未发布 staging 被保留且不会递归自动删除；重复失败可能累积这些目录。
- [ ] 服务端已提交恢复后，分别模拟活动简历读取和单份简历备份读取失败；两种情况都显示 `数据已恢复` 和 `重试刷新`。
- [ ] 点击 `重试刷新` 只重复读取活动简历及其单份简历备份，不请求 `/api/resumes`，也不再次调用恢复接口。
- [ ] 在上述读取成功后单独模拟草稿预览请求失败；界面使用已有的 `草稿预览失败` 状态，恢复对话框不保持或重新打开，且不再次调用恢复接口。
- [ ] 刷新成功后对话框关闭、焦点返回数据管理入口，正式输出仍为待生成。

## Restart And Regressions

- [ ] 恢复后停止并重新启动同一临时数据根上的服务器，恢复后的注册表、活动简历、YAML 和照片仍然有效。
- [ ] 重启后可以切换简历、保存、替换照片、恢复单份简历备份和显式生成输出。
- [ ] ZIP 导出仍只包含活动数据包，不包含 `output/` 或同级 `pre-import` / `pre-restore` 快照。
- [ ] ZIP 检查、确认导入、取消导入和导入前 `pre-import` 保留行为不回归。
- [ ] 页面底部 `最近备份` 仍只列出并恢复当前单份简历 YAML，不会触发整套数据恢复。

## Git And Privacy

- [ ] 浏览器列表、详情、错误、成功消息和恢复 API 请求/响应都不包含测试数据根的绝对路径；请求只提交快照 ID。
- [ ] `git status --short` 不出现数据根、快照、ZIP、照片、真实 YAML、输出或本机配置。
- [ ] `npm run privacy:check` 和 `git diff --check` 通过。
- [ ] 人工验收使用的姓名、联系方式、简历正文和照片全部为合成内容；没有读取、覆盖或提交个人运行数据。
